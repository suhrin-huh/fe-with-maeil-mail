# CSS 의사 요소(Pseudo-elements)란 무엇인가요?

## CSS 의사 요소(Pseudo-elements)
웹 문서를 구성하는 요소들은 보통 태그 단위로 스타일을 지정하지만, 실제로 UI를 구현하다 보면 요소의 전체가 아니라 <u>요소 내부의 특정 부분</u>에만 스타일을 적용해야하는 경우(e.g. 텍스트의 첫 글자만 장식하기, DOM에 존재하지 않는 작은 아이콘이나 장식 넣기)가 자주 생긴다. 이때 HTML 구조를 변경하거나 새로운 요소를 추가해버리면 코드가 지저분해지고 유지보수가 어려워지는데, CSS는 이러한 문제를 해결하기 위해 **의사요소(Pseudo-elements)**라는 개념을 제공한다. 의사 요소는 실제 DOM에 존재하지 않는 가상의 부분에 스타일을 적용하는 방식으로, UI의 표현력을 높이면서도 마크업을 깔끔하게 유지할 수 있게 한다.

## CSS 의사 요소의 종류
의사 요소는 ::before, ::after, ::first-letter, ::first-line처럼 요소 내부에서 특정한 영역을 가리키는 역할을 한다. 특히 ::before와 ::after는 콘텐츠를 삽입할 수 있는 독특한 기능을 제공하는데, 이는 HTML에 새로운 요소를 추가하는 것처럼 보이지만 실제로 문서구조가 변하진 않는다. 다음 예시는 버튼 앞에 "*"을 넣는 매우 흔한 패턴이다. 

```css
button:before {
  content: '*';
  margin-rignt: 6px;
  display: inline-block;
}
```

위의 예시를 통해 버튼 내부의 실제 텍스트보다 앞에 불꽃 이모지를 가상으로 추가할 수 있다. DOM을 열어봐도 <button> 내부에는 여전히 텍스트만 존재한다는 점이 특징이다. 프레임워크를 사용하든 순수 HTML을 사용하든, 마크업 변경 없이 장식을 더할 수 있다는 점은 컴포넌트 UI를 관리할 때 매우 큰 장점이다. 같은 방식으로 ::after를 활용하면 요소 뒤쪽에도 장식을 배치할 수 있다.

텍스트 기반 UI에서도 의사 요소는 유용하다. 예를 들어 본문 첫 줄만 강조하는 잡지 스타일의 디자인을 만들고 싶을 때는 ::first-line을, 첫 글자를 드롭캡(drop cap) 형태롤 만들고 싶을 때는 ::first-letter를 사용할 수 있다.

```css
.article::first-line {
  font-weight: bold;
  color: #333;
}

.article::first-letter {
  font-size: 2.4rem;
  float: left;
  margin-right: 6px;
}
```

이러한 기능 덕분에 구조와 표현을 깔끔하게 분리할 수 있고, 유지 보수성과 시맨틱 구조가 더 강해진다.


## 의사 클래스(Pseudo-classes)

그렇다면 의사 요소와 자주 혼동되는 의사 클래스(Pseudo-classes)는 어떤 차이가 있을까? 두 개념은 모두 선택자 뒤에 붙는 점에서 비슷하지만, 초점을 두고 있는 대상이 다르다.
의사 요소는 요소의 특정 부분에 스타일을 적용하는 반면, 의사 클래스는 요소의 상태나 조건을 기반으로 스타일을 변경한다는 점에서 차이가 있다. 사용자가 링크 위에 마우스를 올렸을 때 변화하는 :hover, 폼 입력이 포커스를 받는 순간을 나타내는 :focus, 특정 조건에 맞는 요소를 선택하는 :nth-child() 등이 대표적이다.

```css
button:hover {
  background-color: #ffd966;
}
```

이 예시에서 :hover는 요소의 상태만 판단할 뿐 요소의 내부적인 구조에는 관여하지 않는다. 반대로 의사 요소는 요소의 "부분"을 가리키며, 그 부분이 실제 DOM 노드로 존재하지 않아도 스타일을 적용할 수 있다. 정리하면 다음과 같은 관점에서 이해하면 된다.

> 의사 클래스 -> "요소가 어떤 상태인가"
>
> 의사 요소 -> 요소의 어떤 부분에 스타일을 적용할 것인가?

두 개념은 그 목적이 다르기 때문에 UI를 구현할 때에 상황에 적절히 맞젝 섞어 쓰게 된다.

```css
/* 마우스를 올렸을 때 버튼 뒤에 "good"이 나오기 */
button:hover::after {
 content: "good";
 margin-left: 4px;
}
```

이처럼 의사 요소는 구조를 변경하지 않고도 표현력을 확장하고, 의사 클래스는 요소의 상태 변화에 따른 인터랙션을 구현한다. 프론트엔드에서 두 개념을 정확히 구분하고 활용하면 UI 구조의 복잡도를 낮추고 유지보수를 훨씬 수월하게 만들 수 있다.


## 한줄 정리
### CSS 의사 요소(Pseudo-elements)란 무엇인가요?
의사 요소는 CSS 선택자에 추가하는 키워드로, 특정요소의 일부분에 스타일을 적용할 때 사용합니다. 의사 요소를 사용하면 요소 내부의 특정 부분에만 스타일을 적용할 수 있습니다.

## 의사요소와 의사 클래스의 차이는 무엇인가요?
의사 요소는 요소의 특정 부분에 스타일을 적용하는 반면, 의사 클래스는 요소의 상태나 조건을 기반으로 스타일을 변경한다는 점에서 차이가 있습니다.
