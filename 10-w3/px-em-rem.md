# 길이 단위 px, em, rem에 대해 각각 설명해주세요.

## px(pixel)

px은 화면상의 CSS 픽셀(CSS pixel)을 기준으로 하는 절대 길이 단위이다. 디바이스의 실제 물리적 픽셀과 1:1로 대응하지 않을 수 있으며, 브라우저가 디바이스 해상도(DPR)에 따라 보정하여 표시한다.

### 특징
- 다른 요소나 상위 요소의 속성에 영향을 받지 않으며 항상 동일한 크기를 유지한다.
- 단일 경계선(border)과 같은 세밀한 UI 요소 표현에 적합하다.
- 브라우저의 글자 크기 확대 설정에 의해 비례적으로 변하지 않으므로 가변성이 떨어질 수 있다.

> 고정된 크기를 요구하는 요소, 해상도에 관계없이 일정한 시각적 표현이 필요한 경우에 용이하다.

## em(element em)

em은 현재 요소 또는 부모 요소에 적용된 font-size를 기준으로 계산되는 상대 길이 단위이다. 기본적으로 브라우저의 초기 글꼴 크기가 16px인 경우, 상속 값이 없다면 1em=16px로 해석된다.

### 특징
- 부모 요소의 font-size에 따라 값이 결정되므로, 계층이 깊어질수록 크기가 누적될 수 있다.
- 텍스트 크기 변화에 따라 padding, margin 등 다른 길이 속성도 비례적으로 변한다.
- 컴포넌트 내부 요소들이 동일한 비율을 유지해야 하는 경우에 효과적이다.

> 버튼, 배지 등 텍스트 크기에 따라 전체 레이아웃이 함께 변화해야하는 UI 구성 요소, 상위 요소의 크기 조절만으로 해당 구성 용소가 일괄적으로 스케일 조정이 되도록 만들 때 용이하다.

## rem(root em)

rem은 root(HTML)의 font-size를 기준으로 계산되는 절대적 기반을 가진 상대 단위이다.

### 특징
- 부모 요소의 크기 변화와 상관없이 항상 동일한 기준을 사용한다.
- HTML의 font-size만 조절해도 전체 사이트의 비례적 크기 변경이 가능하다.
- 브라우저의 글꼴 크기 확대 설정을 자연스럽게 반영하여 사용자 친화적이다.

> 전체 사이트의 타이포그래피 체계 설계, 일관된 spacing 시스템 구축, 디자인 시스템 또는 컴포넌트 라이브러리 설계 시 기반 단위로 사용된다.

## 상황별 권장 단위 선택

|단위|px|em|rem|
|----|----|----|----|
|고정된 크기를 요구하는 경우 <br> 단일 경계선과 같은 미세 UI 효과|텍스트 크기에 따라 레이아웃 전체가 자연스럽게 비례 조정되어야 하는 컴포넌트 <br> 상위 요소의 글꼴 크기만 조절해도 전체가 함께 축소되길 원하는 경우| 사이트 전체의 크기 정책을 일관되게 유지해야 하는 경우|

## Q. 하나의 프로젝트에는 하나의 단위만 사용해야하나요?

**A. NO**  

하나의 CSS 단위가 모든 상황을 해결할 수는 없기에 프로젝트의 규모가 커질수록 적절한 단위 선택이 유지보수성과 접근성, 디자인 일관성에 직접적인 영향을 미친다. 따라서 역할을 기준으로 명확한 규칙을 두고 px, em, rem을 목적에 따라 조합하여 사용하는 것이 좋다.
